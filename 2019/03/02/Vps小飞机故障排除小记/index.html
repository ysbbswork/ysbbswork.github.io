Vps小飞机故障排除小记

发表于 2019-03-02 | | 阅读次数

我的chrome默认搜索时Google，小飞机出故障了几乎瞬间我就能感知到难受，

像这种情况一般更新一下PAC就可以解决，但是这次更新PAC的时候提醒：

Failed to update PAC file

这种情况还是没遇到过，

首先排查一下是不是客户端的问题，发现手机端和其它多个平台上都用不了了，

排除客户端故障。

怀疑是自己的Vps到期了，Xshell远程连了一把，Vps是online的，

应该没问题，而且上了后台查看一下，还有十几天才是下个季度的付费时间，

所以排除Vps停止服务。

可能是进程挂了，重启了一下小飞机的服务进程。

发现还是没有用。

真的很无奈。

没办法，没有小飞机，Google用不了，

只能先百度一下喽，无果。

抱着希望bing一下，还是无果。

突然发现我以前装过larten，免费的梯子，但是很不稳定，不过上google是没问题的，然后google了一下，在前几条就找到了

“尝试修改服务器端口，并重启服务试试”。

试了一下，并更新本地PAC，成功，我的小飞机又飞起来了。

后记：个人感觉这种情况是服务端口被长城ban了，所以换一个端口，就可以了。

网上也有类似言论：“简单说一下，360 参与了 GFW 的升级和维护工作，三六零也是国家重点网络安全科技企业。什么意思自己琢磨一下，所以绝对不建议用任何 360 的产品！”

“电脑上有安装 360 安全管家和360软件的如 360 浏览器，360 杀毒的，请自行卸载 360 相关的软件，具体的原因不多说了，之前已经说过很多次。”
